🎯 Что это

«Коробка с готовой IT-командой» для Claude Code. Подключаешь к проекту — и у Claude появляются роли (агенты), сценарии (скиллы/команды) и реальные действия (MCP-инструменты). Claude перестаёт быть «одним помощником» и становится оркестратором команды. Форк ruvnet/claude-flow (MIT).

🔄 Как работает (главный принцип)

«Команда координирует — Claude Code создаёт». На сложной задаче Claude поднимает параллельных агентов-специалистов (Task-tool), они общаются напрямую и передают работу по конвейеру, Claude сводит результат:
🔍 researcher→🏗️ architect→💻 coder→🧪 tester→👀 reviewer
Маршрутизация: regex по ключевым словам + моё суждение. Анти-дрейф: иерархия, 6–8 агентов, чёткие роли. Tier 0 боссы → 1 топологии → 2 консенсус → 3 исполнители.

⚡ Два главных сценария

🔍 /analyze-project
Аудит существующего репо (или git-URL → клон): уязвимости, сборка, тесты, узкие места → отчёт P1/P2/P3 + план.
🆕 /new-project
Сборка нового с нуля: идея → спека → архитектура → код → тесты.

Verify-гейт: «готово» = собирается + тесты проходят + реально запускается. Проверено вживую: PR #9 и #10 на реальном Playwright-репо — смержены.
