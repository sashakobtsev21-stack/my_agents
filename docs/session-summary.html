1 · Честность и доверие

Сфабрикованные перф-цифры («150x-12,500x», «2.49x-7.47x») были по всему репозиторию выданы за факт. Вычищены построчно в рантайм-выводе CLI, ~240 доках/скиллах/агентах, исходниках, конфиге — честные дисклеймеры и исторические ADR не тронуты. Полые плагины (gastown, code-intelligence, legal-contracts) помечены experimental. Закрыта критическая CVE (vitest → 4.1.8).

2 · Новые first-class сценарии

Добавлены два скилла со встроенным verify-гейтом («готово» = собирается + тесты + запуск): /analyze-project (аудит любого репо → отчёт P1/P2/P3 + план) и /new-project (идея → рабочая первая версия). Скиллов стало 41. Плюс docs/CORE-AGENTS.md — курируемое ядро ~20 агентов вместо разбора всех 121.

4 · Декомпозиция god-файлов и типы

hooks.ts 5331→5072 и hooks-tools.ts 4600→4510 — выделены cohesive-модули с тестами. Убраны все catch (e: any) в v3 → узкие типизированные касты.

5 · Кросс-платформа и инфраструктура тестов

  • Структурная сирота v3/mcp (без package.json) → дан манифест + workspace-член: ~268 тестов резолвят zod и зеленеют (падали и в CI).
  • 2 реальных cross-platform бага в коде: plugins safePath (drive-letter на Windows), file-organization (path.posix.join — консистентный /).
  • POSIX-only тесты → платформо-агностичные; statusline-drift регенерирован; vitest перестал гонять застывшие копии из node_modules (475→210 файлов).
Остаток падений (~20 файлов) — это native-деки (ruvector/sona/ruvllm/onnx/gguf/rvf) и docker: их нет на Windows-хосте, в CI (Linux+native) они зелёные. Это окружение, не код.

6 · Пилот на реальном репо

Сценарий /analyze-project прогнан end-to-end на стороннем Playwright-репо: аудит (3 параллельных специалиста + npm audit/typecheck/lint) → отчёт → фиксы (утечка ключа в телеметрию, CI-гейт visual/perf, env-валидация, serial-cleanup, prettier) → Pull Request → смержен. Доказали, что флоу «анализ → фикс → PR» реально работает.

⚙️ Как проект работает сейчас

my_agents — переиспользуемая AI-команда для Claude Code (форк ruvnet/claude-flow, MIT): 121 агент · 41 скилл · 168 команд · 33 плагина · ~313 MCP-инструментов в монорепо из 24 v3-пакетов.
  • Claude Code = оркестратор + исполнитель: читает файлы, гоняет реальные тулзы (build, npm audit, тесты, git), делегирует параллельным агентам-специалистам и сводит результат.
  • Агенты / скиллы / команды — текст-инструкции (кем быть / как делать / ярлык). MCP-инструменты — код-действие. Плагины — упаковка, которая всё привозит.
  • Два главных флоу: /analyze-project (аудит чужого репо) и /new-project (сборка нового) — оба с verify-гейтом.
  • Статуслайн в Claude Code показывает живую телеметрию (домены, сворм, CVE, память, тесты) — расшифровка в README.
Состояние: CLI собирается, npm audit 0 уязвимостей, ~200+ новых тестов зелёные. Тяжёлые native-тесты зелёные в CI. Всё в origin/main.
