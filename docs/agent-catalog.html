🎖 Оркестраторы, координаторы и руководители

АгентТирНаправлениеКогда подключать
queen-coordinator 🎖 ✓opusHive-MindСуверенный оркестратор hive (Tier 0). Использовать как верхнеуровневого лида запуска hive-mind — владеет целью, выбирает топологию, распределяет ресурсы между классами агентов и удерживает согласованность hive. Выдаёт авторитетный стратегический план.
collective-intelligence-coordinator 🎖 ✓opusHive-MindУзел коллективного интеллекта (Tier 0). Использовать для оркестрации распределённого мышления внутри hive — агрегация решений, синхронизация общих знаний и балансировка когнитивной нагрузки. Выдаёт согласованное коллективное решение и синхронизированный граф знаний.
project-coordinator 🎖 ✓sonnetПрочиеКоординирует мультиагентные workflow — декомпозирует цель, назначает именованных агентов, выстраивает передачи через SendMessage и синтезирует результаты. Использовать как лида для многошаговых, мультиагентных задач.
dual-orchestrator 🎖 ✓opusDual-Mode (Claude + Codex)Оркестрирует Claude Code (интерактивный) и Codex (headless) для гибридных рабочих процессов.
task-orchestrator 🎖 ✓haikuTemplatesЦентральный координирующий агент для декомпозиции задач, планирования выполнения и синтеза результатов.
sparc-coord 🎖 ✓sonnetTemplatesОркестратор методологии SPARC для систематической координации фаз разработки.
adaptive-coordinator 🎖 ✓sonnetSwarm (топологии)Координатор адаптивной топологии. Использовать, когда оптимальная форма swarm заранее неизвестна и должна переключаться по живым метрикам — выбирает hierarchical/mesh/ring/hybrid, отслеживает производительность и безопасно мигрирует с откатом. Выдаёт активную топологию и план миграции.
hierarchical-coordinator 🎖 ✓sonnetSwarm (топологии)Иерархический (queen-led) координатор swarm. Использовать, когда сложная задача требует централизованного планирования с одним координатором, делегирующим специализированным воркерам — анти-дрейфовый дефолт для кодинговых swarm. Выдаёт дерево задач, карту назначения агентов и интегрированный результат.
mesh-coordinator 🎖 ✓sonnetSwarm (топологии)Координатор peer-to-peer mesh-swarm. Использовать, когда агенты должны сотрудничать на равных без единой точки отказа — отказоустойчивое, устойчивое к разделению, распределённое принятие решений. Выдаёт назначения пиров и консенсусные решения.
byzantine-coordinator 🎖 ✓sonnetConsensus / распределённыеКоординатор византийски-отказоустойчивого консенсуса. Использовать, когда узлы могут быть враждебными/скомпрометированными и нужно согласие несмотря на произвольные сбои (выдерживает f < n/3). Выдаёт зафиксированное по PBFT значение и отчёт о вредоносных участниках.
codex-coordinator 🎖 ✓sonnetDual-Mode (Claude + Codex)Координирует несколько headless Codex-воркеров для параллельного выполнения.
crdt-synchronizer 🎖 ✓sonnetConsensus / распределённыеКоординатор синхронизации состояния через CRDT. Использовать, когда несколько писателей (включая офлайн) одновременно меняют общее состояние и оно должно сходиться автоматически без координации. Выдаёт слитое, бесконфликтное состояние.
game-director 🎖 ✓opusGame Dev (Unity / 3D mobile)Креативный и технический директор игры — владеет видением, столпами и Game Design Document, удерживая согласованность всех дисциплин. Использовать как Tier-0 лида для всего игрового проекта.
gossip-coordinator 🎖 ✓sonnetConsensus / распределённыеКоординатор gossip/эпидемического консенсуса. Использовать для масштабного, высокодоступного распространения, где итоговая согласованность важнее строгого согласия. Масштабируется на множество узлов с вероятностными гарантиями.
memory-coordinator 🎖 ✓haikuTemplatesУправляет персистентной памятью между сессиями и обеспечивает обмен памятью между агентами.
performance-benchmarker 🎖 ✓sonnetConsensus / распределённыеСпециалист по бенчмаркингу консенсуса. Использовать для измерения и сравнения протоколов консенсуса (throughput/latency/отказоустойчивость) под вашей реальной нагрузкой, чтобы выбирать между ними эмпирически. Сам по себе не стратегия консенсуса.
pr-manager 🎖 ✓sonnetGitHubВладелец жизненного цикла pull request. Использовать для ведения ОДНОГО PR от начала до конца — создание, метаданные, ревьюеры, метки, согласование CI/тестов, разрешение конфликтов и merge. Канонический владелец жизненного цикла PR.
quorum-manager 🎖 ✓sonnetConsensus / распределённыеКоординатор динамического кворума и членства. Использовать, когда нужны настраиваемые кворумы чтения/записи для тонкой настройки компромисса согласованность-vs-доступность без перехода к полному лидерству Raft.
raft-manager 🎖 ✓sonnetConsensus / распределённыеКоординатор консенсуса Raft. Использовать, когда узлы доверенные (только crash-сбои) и нужен единый авторитетный лидер с линеаризуемым реплицированным логом. Выдерживает f < n/2 crash-сбоев. Дефолт для hive-mind.
release-manager 🎖 ✓sonnetGitHubПланировщик и координатор релизов. Использовать для ПЛАНИРОВАНИЯ одного релиза — решение по semver, changelog/release notes, выравнивание версий между пакетами, гейты валидации и последовательность/откат — завершающееся релизным PR/тегом.
scout-explorer 🎖 ✓sonnetHive-MindСпециалист по разведке (Tier 3). Использовать для исследования неизвестной территории — построить карту кодовой базы/зависимостей, просканировать окружение, выявить угрозы и возможности — и доложить разведданные в hive. Только собирает и докладывает; никогда не меняет код.
security-manager 🎖 ✓sonnetConsensus / распределённыеСлой безопасности консенсуса. Использовать, чтобы добавить криптографическую аутентификацию, контроль членства и обнаружение атак (Byzantine/Sybil/Eclipse/DoS) поверх стратегии консенсуса — не самостоятельный консенсус.
swarm-memory-manager 🎖 ✓haikuHive-MindСлой памяти/персистентности hive (Tier 3). Использовать для управления распределённой памятью — согласованность, персистентность, быстрый поиск, кэширование и синхронизация — долговечный субстрат, который читает/пишет каждый агент. Механический и частый.
sync-coordinator 🎖 ✓sonnetGitHubКоординатор выравнивания нескольких пакетов. Использовать для постоянного выравнивания между пакетами/репозиториями — гармонизация версий, разрешение зависимостей, синхронизация docs/config — и итогового sync-PR. Удерживает согласованность во времени.
worker-specialist 🎖 ✓sonnetHive-MindСпециалист по выполнению задач hive (слой исполнения). Использовать для выполнения конкретной назначенной задачи с точностью, отчитываясь о прогрессе до/во время/после. Выдаёт завершённый продукт работы, никогда не стратегию.
