⚙️ Как это работает

Claude Code выступает ведущим: разбивает задачу и через SendMessage раздаёт её именованным агентам. Те работают параллельно и передают результат дальше по конвейеру. Координаторы (🎖) держат топологию и консенсус, специалисты исполняют, память (AgentDB/HNSW) хранит общий контекст.

User
 └─ Claude Code (lead)
     ├─ 🎖 coordinator      queen · hierarchical · mesh · adaptive
     │    ├─ researcher → architect → coder → tester → reviewer   (конвейер)
     │    ├─ 🎖 consensus    raft · byzantine · quorum · gossip · crdt
     │    └─ memory-manager  AgentDB + HNSW  (общий namespace)
     └─ синтез результата ←┘

Тиры моделей: opus — архитектура/безопасность/сложное · sonnet — основная работа · haiku — простое/механическое. Каждый агент объявляет свой тир в секции Model & cost — дорогая модель тратится только там, где реально нужна.
