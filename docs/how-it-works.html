⚙️ Что и как работает

AlexKo (репозиторий my_agents) — переиспользуемая AI-команда для Claude Code. Как она устроена и как ею пользоваться.

124 агент · 41 скилл · 168 команд · 33 плагина · ~313 MCP-инструментов · 24 v3-пакета · форк ruvnet/claude-flow (MIT)

1 · Что это, в двух словах

Это «коробка с готовой IT-командой» для Claude Code. Ты подключаешь её к своему проекту — и у Claude появляется набор ролей (агенты), сценариев (скиллы/команды) и реальных действий (MCP-инструменты), плюс координация между ними. Claude перестаёт быть «одним помощником» и становится оркестратором команды.

2 · Строительные блоки (главное различие)

БлокЧто этоПрирода
Агентроль: «кем быть» (coder, security-auditor, reviewer…). 124 шт.текст-инструкция
Скилл / Командасценарий: «как делать» / ярлык (/analyze-project). 41 скилл.текст-инструкция
MCP-инструментдействие: реально что-то сделать (записать память, запустить сворм). ~313 шт.код-действие

Ключ: агент/скилл/команда — это инструкции (текст). MCP-инструмент — это код, который реально действует. Плагин — то, что всё это доставляет.

3 · Как Claude Code этим управляет

Главный принцип: «команда координирует — Claude Code создаёт».
  • Claude Code = оркестратор + исполнитель. Он читает файлы, гоняет реальные тулзы (build, npm audit, тесты, git), пишет код — и делегирует подзадачи параллельным агентам-специалистам через Task-tool.
  • CLI / MCP = координация. Инициализация сворма, выбор топологии, общая память, маршрутизация — это «штаб». Саму работу делают агенты Task-tool.
  • Связь между агентами — SendMessage. Именованные агенты общаются напрямую (pipeline: architect → coder → tester → reviewer), а не только через лида.
🔍 researcher→🏗️ architect→💻 coder→🧪 tester→👀 reviewer
Анти-дрейф по умолчанию: иерархическая топология, 6–8 агентов, чёткие роли (specialized), consensus raft.

5 · Два главных рабочих флоу

🔍 /analyze-project — аудит существующего репозитория

Указываешь репо → 3 параллельных специалиста (безопасность / качество / архитектура) + реальные тулзы (npm audit, typecheck, lint, тесты) → отчёт с приоритетами P1/P2/P3 и планом. Опционально — фиксы + PR.
репо→параллельный аудит→отчёт P1/P2/P3→фиксы → PR

🆕 /new-project — сборка нового продукта с нуля

Идея → требования → архитектура → реализация → тесты → рабочая первая версия.
Verify-гейт у обоих: «готово» = собирается + тесты проходят + реально запускается. Не «по словам».

6 · Память и «интеллект» (честные цифры)

  • AgentDB + HNSW — векторная память с семантическим поиском. Измерено: ~1.9x при N=20k, ~3.2x–4.7x при N=5k против полного перебора (recall@10 ~0.99); ANN выигрывает выше точки перелома.
  • SONA — само-оптимизация (измерено 0.0043ms/adapt) · MoE — маршрутизация экспертов · EWC++ — против забывания.
  • Пайплайн: RETRIEVE (HNSW) → JUDGE (вердикты) → DISTILL (LoRA) → CONSOLIDATE (EWC++).
⚠️ Старые рекламные цифры «HNSW 150x–12,500x» и «Flash Attention 2.49x–7.47x» — не воспроизведены / без бенчмарка. В проекте заменены на измеренные или помечены «unverified».

7 · Статуслайн в Claude Code

Внизу AlexKo рисует живую панель (скрипт .claude/helpers/statusline.cjs), нарочно лаконичную — только полезное: модель, % контекста (когда закругляться), и строка статуса — сворм (идёт ли параллельная работа), CVE (безопасность), тесты. Это барометр, а не баннер. Полная расшифровка — в README.

8 · Как подключить к своему проекту

  • Плагины Claude Code (рекомендуется) — установить из этого репозитория в сессии Claude Code.
  • Копирование .claude/ — положить агентов/скиллы в целевой проект.
  • npx alexko … (алиас npx ruflo …) — публичный npm-пакет CLI (общий движок, не твоя кастомная команда).
Чтобы подключить именно мою команду агентов/скиллов — способ 1 или 2 (npm-пакет ставит общий движок). Детали — в README.
